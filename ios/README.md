# Longevity Stack — iOS (SwiftUI + HealthKit)

A native iOS port of the [web Longevity Stack](../longevity/), adding the one
thing a web app can't do: **Apple Health integration**.

## What HealthKit adds

| Capability | How it's used |
|---|---|
| **Read steps** | "Motion (Walk with Earl)" auto-completes at an adjustable step goal (default **6,000**, tunable in-app). |
| **Read active energy** | Today's kcal shown in the footer. |
| **Show steps** | A live "Steps Today" stat tile. |
| **Write Mindful Minutes** | Checking **Kumbhaka Breathwork** logs a 5-minute mindfulness session to Apple Health. |

## Other features

| Feature | Notes |
|---|---|
| **Daily reminder** | A single repeating local notification at a time you pick. No server/push cert — pure `UserNotifications`. |
| **Fasting log** | Every completed fast (≥1 min) is recorded and shown in a "Fasting Log" card, color-coded against your goal. Stored locally — see the note below. |
| **30-day charts** | Swift Charts card showing pillar completion and fasting hours per day for the last 30 days, with your fasting goal drawn as a reference line. |
| **Home-screen widget** | Small + medium WidgetKit widget showing live fast progress, pillar count, and streak. |
| **Lock Screen widgets** | Circular (fast gauge), rectangular (timer + pillars + streak), and inline accessory widgets for the iOS Lock Screen. |
| **Adjustable step goal** | Tune the Motion auto-complete threshold (1k–20k) right in the app. |

Everything else (fasting ring, pillar checklist, streak, best-fast) mirrors the
web app. State is persisted in a shared **App Group** (`group.com.cjaynyc.longevitystack`)
so the widget can read it.

> **Why the fasting log isn't written to Apple Health:** HealthKit has no
> fasting data type — there's no `HKObjectType` for fasting. Writing it as a
> Workout or Mindful sample would corrupt your real fitness data, so the fast
> history is kept inside the app instead. (If Apple adds a fasting type later,
> this can mirror to Health.)

## Requirements

- A **Mac with Xcode 15+**
- An **iPhone** (HealthKit doesn't return data in the Simulator — auto-complete
  needs a real device with Health data)
- An Apple ID. A free one lets you side-load to your own iPhone for 7 days;
  a paid **Apple Developer account ($99/yr)** is needed for permanent installs
  / TestFlight / the App Store.

## Build it

### Option A — XcodeGen (recommended, generates the project for you)

```bash
brew install xcodegen      # once
cd ios
xcodegen generate          # creates LongevityStack.xcodeproj + Info.plist + entitlements
open LongevityStack.xcodeproj
```

In Xcode: select the **LongevityStack** target → **Signing & Capabilities** →
choose your Team (Xcode adds a free provisioning profile). Plug in your iPhone,
pick it as the run destination, and press ▶.

### Option B — Manual Xcode project (no XcodeGen)

1. **File ▸ New ▸ Project ▸ iOS ▸ App**, name it `LongevityStack`,
   interface **SwiftUI**, language **Swift**.
2. Delete the auto-generated `ContentView.swift` / `App.swift`, then drag every
   file from `Sources/` into the app target (check *Copy items if needed*).
3. **Signing & Capabilities** for the app target → **+ Capability**:
   - **HealthKit**
   - **App Groups** → add `group.com.cjaynyc.longevitystack`
4. In the app target's **Info** tab add these two keys:
   - `Privacy - Health Share Usage Description` → *"Longevity Stack reads your
     step count and active energy to auto-complete your daily Motion pillar."*
   - `Privacy - Health Update Usage Description` → *"Longevity Stack logs your
     Kumbhaka breathwork as Mindful Minutes in Apple Health."*
5. **Add the widget:** **File ▸ New ▸ Target ▸ Widget Extension**, name it
   `LongevityWidget` (uncheck "Include Configuration Intent"). Replace its
   generated file with `Widget/LongevityWidget.swift`, and add
   `Sources/Shared.swift` + `Sources/Theme.swift` to the widget's target
   membership. Give the widget target the **same App Group** capability.
6. Run on your iPhone. On first launch, approve the Health permission sheet;
   enable the daily reminder to grant notifications. Long-press the home screen
   to add the widget.

> The `*.plist` / `*.entitlements` files are generated by XcodeGen (Option A).
> With Option B, Xcode manages those for you — you don't create them by hand.
> The key manual step is giving **both** targets the same App Group so the
> widget can read the app's data.

## Source layout

```
ios/
├── project.yml                 # XcodeGen config (both targets, plists, entitlements)
├── README.md
├── Sources/                    # app target
│   ├── LongevityStackApp.swift # @main entry point
│   ├── AppState.swift          # fasting + pillars + streak + step goal, persisted
│   ├── HealthManager.swift     # HealthKit auth, reads, mindful-minute writes
│   ├── NotificationManager.swift # daily local reminder
│   ├── HistoryView.swift       # 30-day Swift Charts card
│   ├── Shared.swift            # App Group store + snapshot (shared with widget)
│   ├── Theme.swift             # colors + Card container (shared with widget)
│   └── DashboardView.swift     # the UI
└── Widget/                     # widget extension target
    └── LongevityWidget.swift   # Home Screen + Lock Screen widgets
```

## Notes & next steps

- HealthKit returns no data in the iOS Simulator — test on a device, or add
  sample steps in the Health app first.
- The widget reads a snapshot written by the app, so it updates after you open
  the app at least once (and ~every 15 min thereafter; the live fast clock
  ticks on its own).
- Possible follow-ons: a 30-day history chart, a Lock Screen widget, mirroring
  fasts to Health if Apple adds a fasting type, or an Apple Watch companion.
